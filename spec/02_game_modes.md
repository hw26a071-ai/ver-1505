# ゲームモード・スコア仕様書 (Game Modes & Score)

## 1. 各コースの詳細

### 1.1. チュートリアル
- **目的**: プレイヤーにゲームの操作方法を理解させ、カメラの認識環境を最適化する。
- **主な機能**:
  - ゲームの全体概要とルールの説明。
  - マウス、キーボード、ハンドトラッキングの入力テスト。
  - トラッキング精度の確認・調節（カメラ位置の調整、手の認識テスト等）。

### 1.2. 手話ジェスチャーコース
- **目的**: 提示されたお題の手話（ジェスチャー）を正確に行う。
- **ゲームルール**:
  - **超コンパクト・スクロールレス設計**: 手本（イラスト・画像・動画のプレースホルダー）を完全に消去し、カメラプレビューの大きさを従来の半分ほど（横幅を抑えたコンパクトサイズ）に縮小。カメラのすぐ周辺に解答開始ボタン、現在のカウントを表示するバー、お題テキストを配置し、一切のスクロールなしで1画面内で完結してゲームプレイができる、快適で洗練されたUXを実現。
  - **解答開始ボタンの導入**: お題が提示された段階では自動的に判定は始まらず、プレイヤーが初期姿勢や手構えを整えてから、自身のタイミングで**「解答開始ボタン」**を押します（これにより、手が映り込んだだけで勝手にクリアする誤判定を完全に防止します）。
  - **解答前カウントダウン**: 解答開始ボタンが押されると、画面上で**3秒間のカウントダウン**が開始されます。この間にプレイヤーはカメラの前で手話の開始姿勢（初期ポーズ）を準備します。
  - **3秒間の解答タイム**: カウントダウン終了と同時に、**3秒間の解答タイム（計測・判定時間）**が開始されます。プレイヤーはこの3秒間の間に、実際の手話の動き（一連の流れ）を行います。
  - **AI学習データとの一致判定**: 3秒間の解答タイム中にカメラが捉えた手の軌跡・指の動きを時系列で計測し、**エディットモードの「AIトラッキング学習」によって保存された学習データ（一連の流れ）**と照合します。
  - 解答タイム中の動きと、学習データの動きが一致した場合にのみ「正解（クリア）」と判定され、1ポイント獲得となります。不一致の場合は判定失敗となり、再度「解答開始ボタン」を押して最初からやり直す制御となります。

### 1.3. 読み取りタイピングコース
- **目的**: 提示された手話の意味を理解し、正しくタイピングする。
- **ゲームルール**:
  - 画面に手話の見本（動画、またはアニメーション等）が提示される。
  - プレイヤーはその手話の意味を判別する。
  - キーボードで答え（読み）をタイピングし、エンターキー（決定ボタン）を押して決定します。
  - タイプミス（入力途中の誤字をバックスペース等で修正すること）による減点や失敗ペナルティは発生しません。
  - エンターキーで決定した回答が間違っていた場合に「失敗」と判定されます。

## 2. 難易度選択 (レベル1〜レベル5)
プレイヤーの習熟度に合わせて難易度を5段階から選択します。手首や指のイラスト手本を完全に廃止したため、ヒントとして「手話の具体的な動きに関するテキスト説明（動作ヒント）」を画面に表示するかどうかで難易度を制御します。

| 難易度レベル | お題の難易度 | 動作ヒントの表示 (テキスト) | 制限時間 | 特殊ペナルティ・ルール |
| :--- | :--- | :--- | :--- | :--- |
| **レベル1** | 標準 (ランダム選出) | あり (詳細な動きを常に表示) | なし | 特になし（標準の学習環境） |
| **レベル2** | 上昇 (ランダム選出) | あり (詳細な動きを常に表示) | なし | 特になし |
| **レベル3** | 上昇 (ランダム選出) | あり (詳細な動きを常に表示) | **あり (1問につき30秒)** | 特になし（時間内のクリアが必要） |
| **レベル4** | 高い (ランダム選出) | **なし** (ヒント非表示) | **あり (1問につき30秒)** | 動作ヒントが表示されないため、手話の記憶と素早い判断が必要 |
| **レベル5** | 高い (ランダム選出) | **なし** (ヒント非表示) | **あり (1問につき15秒)** | **失敗（※）するたびに、現在の獲得スコアが半分に減少する** |

### ※ レベル5における「失敗」の定義
- **手話ジェスチャーコース**: 制限時間（15秒）以内に、指定された正しい手話を完了・認識させることができなかった場合。
- **読み取りタイピングコース**: 制限時間（15秒）以内にエンターキーで回答を決定できなかった場合、またはエンターキーを押したときの決定回答が「不正解（誤答）」だった場合。（タイピング途中での文字の打ち間違いは、時間内であればバックスペース等で修正可能なため「失敗」とはみなしません）

## 3. スコア・プレイシステム
- **出題数**: 1回のプレイにつき **5問**
- **想定プレイ時間**: 1分〜2分
- **ローカルハイスコア**: プレイ終了時、獲得したスコアがハイスコアを更新した場合、ローカルストレージ（Local Storage等）に保存される。ハイスコアは**「コースごと ✕ 難易度レベルごと」**に個別に最高スコアを記録・管理・表示します。

### 3.1. 得点基準
各コースのプレイ結果に応じて、以下の基準でポイントを加算します。

| コース | 加算条件（1問正解ごとに1ポイント） |
| :--- | :--- |
| **手話ジェスチャーコース** | 指示された内容を手話で正しく表現できた場合 |
| **読み取りタイピングコース** | 指示された内容を正しくタイピングし、エンターキーで決定して正解だった場合 |

## 4. エディットモード（AIトラッキング学習）
- **目的**: プレイヤー自身の手の大きさや関節の可動域に合わせて、お題の手話（「こんにちは」「ありがとう」「さようなら」「だめ」「いいよ」「すみません」等）を実際にカメラの前で実演して学習させ、判定アルゴリズムを最適化・チューニングする。
- **画面構成（1画面完結型）**:
  - **スクロールレス設計**: カメラプレビュー、登録状態、学習操作ボタン群、単語選択リストを画面内にコンパクトに一括配置し、スクロールなしで1画面で直感的に操作を完結できるレイアウトにする。
- **主な機能**:
  - **対象単語の選択**: 学習したい手話単語をリストから選択。
  - **リアルタイムサンプリング（時系列動作学習）**: 
    - 「学習開始（データ登録）」ボタンを押すと、画面に**3秒間のカウントダウン（準備時間）**が表示されます。
    - カウントダウン終了後、自動的に**3秒間のデータサンプリング（一連の手話の動きの記録）**を開始します。
    - 3秒間で関節の3D軌跡・指の角度変化を連続サンプリングし、キーフレーム状態遷移判定用のデータとして登録します。
  - **履歴操作（一つ前の学習を忘れる）**: 直前に登録した1つの時系列学習データ（3秒間のシークエンスデータ）のみを消去してサンプリングをやり直せる「1ステップのアンドゥ（一つ前の学習を忘れる）」機能を搭載。
  - **特徴量データの保存（GitHub連携）**: 
    - サンプリングした一連の手話の動き（3秒間の時系列特徴量）を判定用の学習特徴量として保存。
    - 保存先はブラウザの `LocalStorage` に加え、**GitHub連携（個人用アクセストークンやリポジトリパス等の設定）を介して、GitHub上にJSONファイル等として学習データを永続保存・インポート/エクスポートできる仕組み**を提供する。
  - **ゲームプレイへの即時同期適用**:
    - エディットモードでプレイヤーが新しくサンプリングした3秒間の学習データは、**即座に「手話ジェスチャーコース」のゲームプレイ判定用特徴量として同期・適用されます。**
    - 判定用データと学習データは完全に一元化され、問題に挑戦する際にも、プレイヤーが直前までに学習・更新した最新の動作定義に基づいて厳密なキーフレーム状態遷移判定が行われます。
  - **学習リセット機能**: 各単語、またはすべての学習データをリセットし、システムのデフォルト判定基準に戻す。

## 5. 設定項目 (システム設定)
タイトル画面（または専用の設定領域）にて、以下の設定・調整を行うことができます。
- **音量調整**: BGMおよび効果音（正解音、不正解音など）の音量を調整する。
- **設定初期化ボタン**: 変更した音量設定、記録されているローカルハイスコア、および**エディットモードで登録したAIトラッキング学習データ（GitHub/LocalStorage連携データ含む）**を初期状態（工場出荷状態）にリセットする。
